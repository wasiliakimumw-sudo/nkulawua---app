Import Instructions
How to Import Data
  1. Download the appropriate template for the data type you want to import.
  2. Open the template in Excel and fill in your data. Do not change the column headers.
  3. Save the file as .xlsx format.
  4. Select the data type from the dropdown above.
  5. Click "Choose File" and select your Excel file.
  6. Click "Import Data" to upload and process the file.

Fields: invoice_number, client, issue_date, due_date, status, household_count, cost_per_unit, tax_rate, discount, notes

Matching: Records are matched by invoice_number. If an invoice with the same number exists, it will be updated.

client: Enter the exact client name (must exist in system)

status options: draft, sent, viewed, paid, partial, overdue, cancelled

Date format: YYYY-MM-DD (e.g., 2026-01-15)

Fields: expense_number, description, amount, expense_date, vendor, account_code, is_paid

Matching: Records are matched by expense_number. If an expense with the same number exists, it will be updated.

vendor: Enter the exact vendor name (leave empty if none)

account_code: Enter the account code (leave empty if none)

is_paid options: True, False

Date format: YYYY-MM-DD (e.g., 2026-01-15)

Fields: client, amount, payment_date, payment_method, reference, invoice_number

Note: Payments are always created as new records (no update matching)

client: Enter the exact client name (required, must exist)

invoice_number: Enter invoice number to link payment to invoice (optional)

payment_method options: cash, bank_transfer, credit_card, check, other

Date format: YYYY-MM-DD (e.g., 2026-01-15)

Fields: account_code, fiscal_year, amount, notes

Matching: Records are matched by account_code + fiscal_year. If a budget for the same account and year exists, it will be updated.

account_code: Enter the account code (must exist in system)

fiscal_year: Enter a year (e.g., 2026)

Important:
  • The first row of your Excel file must contain column headers matching the template.
  • Data is matched by unique identifiers (name for clients/vendors, code for accounts, etc.).
  • Existing records with matching identifiers will be updated, not duplicated.
  • For transactions (Invoices, Expenses, Payments), related entities must already exist (clients, vendors, accounts).
  • Only the first sheet of the workbook will be processed.
